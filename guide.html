Aperçu

Ce que vous allez recevoir

Chaque jour de bourse, à la clôture, un email soigné atterrit dans votre boîte. Voici ce qu'il contient :

💶

Valorisation & P&L

Montant total, gain/perte du jour et depuis l'achat, en euros et en %.

⏱️

Toutes les périodes

Performance jour, semaine, mois, depuis le 1ᵉʳ janvier et depuis l'ouverture de chaque position.

🥧

Répartition

Graphe en barres de la répartition de votre portefeuille par catégorie (ETF, actions…).

📈

Courbe d'évolution

L'historique de votre valorisation totale, enregistré et tracé automatiquement.

🤖

Note de marché IA

Le vendredi, Claude analyse l'actualité de vos titres et rédige une note (micro / macro / perspective).

📄

PDF + version texte

Le rapport est joint en PDF, avec un repli texte pour tous les clients mail.

💡
Le projet gère un PEA (actions / ETF) et, en option, un PEE (épargne salariale). Le PEE est facultatif : si vous n'en avez pas, mettez simplement "pee": {} dans portfolio.json et tout le bloc PEE disparaît du rapport, qui n'affiche alors que le PEA. À l'inverse, remplissez la section pee (nom du fonds, parts, VL, date de déblocage) pour le voir apparaître.

Installation

1Copier le projet sur votre GitHub

Le code est public. Vous allez en créer votre propre copie (un « fork »).

  1. Connectez-vous à GitHub.
  2. Ouvrez la page de ce dépôt sur GitHub (celui qui contient ce guide).
  3. En haut à droite, cliquez sur Fork, puis Create fork.
  4. Vous obtenez votre copie à l'adresse github.com/VOTRE-PSEUDO/<nom-du-depot>.
🔒
Confidentialité : votre fichier de positions (portfolio.json) contient vos montants réels. Si le dépôt est public, ils sont visibles de tous. Pour les garder privés, créez plutôt un dépôt privé : les minutes GitHub Actions y restent gratuites pour un usage personnel. recommandé

2Renseigner votre portefeuille

Tout se passe dans un seul fichier : portfolio.json. Partez du modèle fourni portfolio.example.json.

Comment éditer un fichier directement sur GitHub

  1. Dans votre dépôt, ouvrez portfolio.example.json, cliquez sur l'icône crayon ✏️ (« Edit »).
  2. Copiez tout le contenu, revenez à la racine, créez un nouveau fichier nommé portfolio.json (bouton Add file → Create new file), collez, puis adaptez.
  3. En bas, cliquez sur Commit changes.

Le format expliqué

// portfolio.json
{
  "portfolio": [
    {"nom": "Amundi MSCI World (CW8)", "ticker": "CW8.PA",  "qte": 10, "pru": 500.00, "cat": "ETF MSCI World"},
    {"nom": "TotalEnergies (TTE)",      "ticker": "TTE.PA",  "qte": 20, "pru": 55.00,  "cat": "Actions FR"}
  ],
  "pee": {
    "parts": 25.0, "pru": 150.00, "vl_last": 165.00, "vl_j2": 164.50,
    "vl_date": "01/01/2026", "disponibilite": "01/01/2030"
  }
}
ChampSignification
nomLibellé affiché dans le rapport (texte libre).
tickerSymbole Yahoo Finance. Pour Euronext Paris, suffixe .PA (ex : TTE.PA). Cherchez sur finance.yahoo.com.
qteNombre de titres détenus.
pruPrix de Revient Unitaire = votre prix d'achat moyen par titre (frais inclus).
catCatégorie pour le graphe de répartition (texte libre, ex : ETF Nasdaq-100, Actions FR).
ℹ️
Le bloc pee est pour l'épargne salariale (fonds à valeur liquidative). Si vous n'avez pas de PEE, laissez l'exemple tel quel ou mettez de petites valeurs - il s'affichera mais sans impact.

3Créer un mot de passe d'application Gmail

Pour qu'un robot envoie un email via votre Gmail, Google exige un « mot de passe d'application » dédié (différent de votre mot de passe habituel).

  1. Activez d'abord la validation en 2 étapes sur votre compte : myaccount.google.com/security (obligatoire pour l'étape suivante).
  2. Allez sur myaccount.google.com/apppasswords.
  3. Donnez un nom (ex : PEA Routine) et cliquez sur Créer.
  4. Google affiche un code de 16 lettres (ex : abcd efgh ijkl mnop). Copiez-le sans les espaces → abcdefghijklmnop.
⚠️
Ce code ne s'affiche qu'une seule fois. Gardez-le de côté pour l'étape 4. Ce n'est pas votre mot de passe Gmail habituel.

4Enregistrer vos identifiants en sécurité

Les mots de passe ne se mettent jamais dans le code. GitHub propose un coffre-fort : les « Secrets ».

  1. Dans votre fork, allez dans Settings (onglet du dépôt, pas celui du compte).
  2. Menu de gauche : Secrets and variables → Actions.
  3. Cliquez sur New repository secret et ajoutez ces secrets un par un :
Nom du secretValeur
GMAIL_USERVotre adresse Gmail expéditrice, ex : monrobot@gmail.com
GMAIL_PASSWORDLe mot de passe d'application à 16 lettres (sans espaces) de l'étape 3.
GMAIL_DESTL'adresse qui reçoit le rapport (peut être la même).
ANTHROPIC_API_KEY optionnelVotre clé API Anthropic (voir étape 5).
🔐
Les secrets sont chiffrés et invisibles, même pour vous, une fois enregistrés. C'est l'endroit sûr pour ce genre d'informations.

5Activer la note de marché IA optionnel

Le vendredi, le projet peut demander à Claude d'analyser l'actualité de vos titres et de rédiger une note. Pour cela, il faut une clé API Anthropic.

  1. Créez un compte sur console.anthropic.com.
  2. Ajoutez un petit crédit (5 € suffisent pour ~1 an). Chaque note coûte ≈ 0,10 à 0,15 €, soit environ 5 à 8 € par an pour 1 note par semaine.
  3. Dans API Keys, créez une clé (commence par sk-ant-...) et copiez-la.
  4. Ajoutez-la dans les Secrets GitHub (étape 4) sous le nom ANTHROPIC_API_KEY.
💸
Détail du coût (modèle Claude Sonnet) : ~2 000 tokens générés (≈ 0,03 €) + le contexte et les 3 recherches web facturées (≈ 0,03 € chacune au maximum). Au total ≈ 0,10 à 0,15 € par note. Comme elle n'est produite qu'une fois par semaine (vendredi à la clôture), cela représente ~5 à 8 € par an - prévisible, et plafonné par votre crédit Anthropic.

6Activer le robot et tester

Le robot est piloté par « GitHub Actions ». On l'active, puis on lance un premier rapport à la demande.

Activer les Actions

  1. Dans votre fork, ouvrez l'onglet Actions.
  2. GitHub désactive les Actions sur les forks par sécurité : cliquez sur « I understand my workflows, go ahead and enable them ».

Lancer un rapport tout de suite (test)

  1. Onglet Actions → dans la liste de gauche, sélectionnez « PEA P&L - Rapport quotidien ».
  2. Cliquez sur Run workflow (à droite) → Run workflow.
  3. Après ~1 minute, le job passe au vert ✅ et l'email arrive dans la boîte GMAIL_DEST.
🔎
Si le job échoue (croix rouge), cliquez dessus pour lire les logs. Causes fréquentes : mot de passe d'application erroné, ticker Yahoo introuvable, ou validation 2 étapes Gmail non activée.

L'envoi automatique quotidien

C'est déjà configuré ! Le fichier .github/workflows/pea_routine.yml contient une planification (« cron ») :

schedule:
  - cron: '30 15 * * 1-5'   # 15h30 UTC, du lundi au vendredi

15:30 UTC ≈ 17h30 à Paris en été (16h30 en hiver), soit après la clôture d'Euronext. Pour changer l'heure, modifiez ce nombre (au format UTC) directement sur GitHub.

7Fiabiliser l'horaire avec cron-job.org optionnel

Le « cron » de GitHub est gratuit mais peut être retardé de 5 à 30 min aux heures de pointe, voire sauté. Pour un envoi à l'heure pile, on délègue le déclenchement à cron-job.org (gratuit), qui « sonne » votre robot via l'API GitHub.

a) Créer un jeton d'accès GitHub (PAT)

  1. Allez sur github.com/settings/tokens → Fine-grained tokens → Generate new token.
  2. Repository access : « Only select repositories » → choisissez votre fork.
  3. Permissions → Repository permissions → Contents : Read and write et Actions : Read and write... puis générez et copiez le jeton (commence par github_pat_...).
🔑
Ce jeton est un mot de passe. Ne le partagez jamais, ne le mettez pas dans le code. Vous le collerez uniquement dans cron-job.org. S'il fuite, révoquez-le immédiatement.

b) Configurer le déclencheur sur cron-job.org

  1. Créez un compte sur cron-job.org → Create cronjob.
  2. URL (remplacez VOTRE-PSEUDO) : 
    https://api.github.com/repos/VOTRE-PSEUDO/pea-routine/dispatches
  3. Réglez l'horaire souhaité (ex : 17h30, du lundi au vendredi, fuseau Europe/Paris).
  4. Dans les réglages avancés, choisissez la méthode POST, puis ajoutez les en-têtes (Headers) : 
    Authorization: Bearer VOTRE_JETON_GITHUB
Accept: application/vnd.github+json
Content-Type: application/json
  5. Dans le corps (Body) de la requête, collez : 
    {"event_type": "run-pea"}
  6. Enregistrez et utilisez « Run now » pour tester : votre robot doit se lancer dans l'onglet Actions.
⚙️
Le workflow écoute déjà cet événement (repository_dispatch: types: [run-pea]). Vous pouvez désormais retirer la ligne schedule: du workflow pour éviter un double envoi, ou la garder en filet de sécurité.
Au quotidien

Mettre à jour vos positions

Quand vous passez un ordre, vous recevez un « avis d'opéré » de votre courtier (quantité, cours, frais). Un formulaire intégré recalcule tout pour vous - pas de maths à faire.

Depuis GitHub (même au téléphone)

  1. Onglet Actions → « Position - Achat / Vente » → Run workflow.
  2. Remplissez le formulaire en recopiant l'avis d'opéré :
    • operation : achat ou vente
    • ticker : ex. TTE.PA
    • quantite, cours, frais (la virgule décimale est acceptée)
    • nom + categorie : uniquement pour une nouvelle valeur
  3. Validez : le robot recalcule le PRU et met à jour portfolio.json tout seul.
OpérationCe qui est recalculé
Achat (renforcement)Nouveau PRU = moyenne pondérée ; quantité augmentée.
Vente partielleQuantité réduite ; le PRU ne change pas.
Vente totaleLa ligne est retirée du portefeuille.
Nouvelle positionAjoutée avec son PRU (nom + catégorie requis).

En local (sur votre ordinateur)

# Renforcer : 10 titres à 57,50 €
python3 position.py achat TTE.PA 10 --cours 57.50 --frais 0

# Alléger : vendre 5 titres
python3 position.py vente TTE.PA 5

# Nouvelle position
python3 position.py achat AI.PA 4 --cours 175.20 --nom "Air Liquide (AI)" --categorie "Actions FR"

Dépannage

Questions fréquentes

Je n'ai pas reçu l'email
Vérifiez vos spams, puis l'onglet Actions : si le job est rouge, lisez les logs. Causes typiques : GMAIL_PASSWORD incorrect (utilisez bien le mot de passe d'application à 16 lettres), validation 2 étapes Gmail non activée, ou un secret mal nommé.
Un de mes titres affiche « N/A »
Le ticker est probablement faux. Cherchez la valeur exacte sur finance.yahoo.com et recopiez le symbole tel quel (avec son suffixe de place, ex. .PA).
Le robot s'arrête après quelques jours
GitHub désactive le cron des dépôts inactifs depuis 60 jours. Faites une petite modification (ou un « Run workflow » manuel) de temps en temps, ou utilisez le déclencheur cron-job.org de l'étape 7 qui n'a pas cette limite.
Combien ça coûte ?
0 € pour GitHub Actions, Yahoo Finance et Gmail. Seule la note de marché IA est payante (clé Anthropic) : ≈ 0,10 à 0,15 € par note, soit ~5 à 8 € par an pour 1 note/semaine - et elle reste totalement optionnelle.
Mes données financières sont-elles publiques ?
Uniquement si votre dépôt est public (les montants de portfolio.json seraient alors visibles). Pour les garder privés, créez un dépôt privé : les Actions restent gratuites pour un usage personnel. Vos mots de passe, eux, sont toujours protégés dans les Secrets.
Je n'ai pas de PEE, c'est grave ?
Non. Laissez le bloc pee de l'exemple, ou mettez de petites valeurs. La section PEE s'affichera mais vous pouvez l'ignorer.
